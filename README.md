# Tetris - 经典俄罗斯方块游戏

一个功能完整的 HTML5 俄罗斯方块游戏，使用纯 HTML5、CSS3 和 JavaScript 构建。采用现代暗色主题，流畅动画和响应式设计。

![Tetris Game](./screenshot.png)

## 特性

### 核心玩法
- **7 种标准方块**: I, O, T, S, Z, J, L - 每种都有独特的颜色
- **方块旋转**: 90 度顺时针旋转，支持墙踢（Wall Kick）
- **幽灵块**: 视觉指示器显示方块将落在哪里
- **消行系统**: 消除完成的行以获得分数
- **等级推进**: 每消除 10 行速度加快
- **下一个方块预览**: 查看接下来会出现的方块

### 计分系统
| 操作 | 基础分数 |
|--------|---------------|
| 单行消除 | 100 |
| 双行消除 | 300 |
| 三行消除 | 500 |
| 四行消除 (Tetris) | 800 |
| 软降 | 每格 1 分 |
| 硬降 | 每格 2 分 |

### 操作控制
| 按键 | 动作 |
|-----|--------|
| ← → | 左右移动 |
| ↑ | 旋转方块 |
| ↓ | 软降（加速下落） |
| Space | 硬降（瞬间到底） |
| P | 暂停/继续 |
| Enter | 开始/重新开始游戏 |

### 额外功能
- **暂停/继续**: 按 P 或使用暂停按钮
- **触摸控制**: 移动设备上支持滑动操作
- **响应式设计**: 桌面和移动端都能完美运行
- **现代 UI**: 暗色主题搭配渐变效果
- **流畅动画**: 消行闪光和升级特效

## 开始使用

### 本地运行

1. 克隆或下载此仓库
2. 在現代浏览器中打开 `index.html`
3. 点击"Start Game"开始游戏！

无需构建过程或依赖 - 只需打开 HTML 文件即可！

### 浏览器支持

- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

## 技术细节

### 游戏架构

游戏使用原生 JavaScript 构建，无需外部框架：

- **游戏循环**: 使用 `requestAnimationFrame` 实现流畅的 60fps 渲染
- **Canvas 渲染**: HTML5 Canvas 提供高性能图形
- **状态管理**: 简单的状态对象进行游戏控制
- **碰撞检测**: 基于网格的碰撞系统，支持墙踢

### 核心组件

#### pieces.js
定义所有方块的形状、颜色和旋转逻辑：
- 每种方块类型的形状矩阵
- 边界附近的旋转算法（墙踢）
- 使用袋子系统的随机方块生成

#### game.js
主要游戏引擎处理：
- 棋盘管理（10x20 网格）
- 移动和碰撞检测
- 消行和计分
- 等级推进
- UI 更新

### 计分公式
```
分数 = 基础分数 × 等级
```
其中基础分数取决于单次操作消除的行数。

## 部署

### GitHub Pages

1. 将代码推送到 GitHub 仓库
2. 进入 Settings → Pages
3. 选择分支和文件夹（通常是 `main` 根目录）
4. 您的游戏将在以下地址可用：`https://username.github.io/repo-name/`

### 手动部署

使用内置的 `deploy.sh` 脚本：

```bash
chmod +x deploy.sh
./deploy.sh
```

这将：
- 创建包含优化文件的 dist 文件夹
- 在 dist 文件夹中初始化 git
- 提交并推送到 gh-pages 分支

## 开发

### 添加功能

代码库设计为易于扩展：

1. **添加自定义方块**: 编辑 `pieces.js` 中的 TETROMINOS 对象
2. **自定义颜色**: 修改 pieces.js 或 CSS 中的颜色值
3. **调整速度**: 更改 game.js 中的 DROP_INTERVAL 或等级推进逻辑

### 代码风格

- ES6+ JavaScript 语法
- 模块化函数设计
- 全面的 JSDoc 注释
- 清晰的关注点分离（pieces 与游戏逻辑）

## 已知问题修复

### v1.1 - 幽灵块重影修复
**修复日期**: 2026-03-26

**问题描述**: 幽灵块在移动时会出现重复渲染的问题。

**解决方案**:
- 使用 `ctx.save()` 和 `ctx.restore()` 保存和恢复 Canvas 状态
- 确保每次绘制后正确重置 `globalAlpha`
- 添加边界检查，避免与实际方块重叠渲染
